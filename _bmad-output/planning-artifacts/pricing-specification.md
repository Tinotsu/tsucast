## Executive Summary

tsucast uses an **article credit** pricing model with no subscription. Users buy credit packs and spend credits to convert articles to audio. Short articles bank leftover time, long articles use multiple credits.

## Implementation Notes

### Database Schema

```sql
-- User credits balance
credits_balance: integer (total minutes available)
credits_purchased: integer (lifetime credits bought)
credits_used: integer (lifetime credits spent)

-- Time bank (for short article rollover)
time_bank_minutes: integer (accumulated from short articles)
```

### Credit Deduction Logic

```typescript
function calculateCreditsNeeded(durationMinutes: number, timeBank: number): {
  creditsNeeded: number;
  newTimeBank: number;
} {
  const MIN_CHARGE = 3;
  const CREDIT_SIZE = 20;

  // Apply minimum
  const effectiveDuration = Math.max(durationMinutes, MIN_CHARGE);

  // Use time bank first
  const netDuration = effectiveDuration - timeBank;

  if (netDuration <= 0) {
    // Covered by time bank
    return { creditsNeeded: 0, newTimeBank: timeBank - effectiveDuration };
  }

  // Calculate credits needed
  const creditsNeeded = Math.ceil(netDuration / CREDIT_SIZE);
  const timeProvided = creditsNeeded * CREDIT_SIZE;
  const newTimeBank = timeProvided - netDuration;

  return { creditsNeeded, newTimeBank };
}
```
